1.0 Prevendo o Nível de Satisfação dos Clientes do Santander¶

A satisfação do cliente é uma medida fundamental de sucesso. Clientes insatisfeitos cancelam seus serviços e raramente expressam sua insatisfação antes de sair. Clientes satisfeitos, por outro lado, se tornam defensores da marca!

O Banco Santander está pedindo ajuda para identificar clientes insatisfeitos no início do relacionamento. Isso permitiria que o Santander adotasse medidas proativas para melhorar a felicidade de um cliente antes que seja tarde demais.

Dataset disponível no Kaggle em: https://www.kaggle.com/c/santander-customer-satisfaction

Objetivo¶

Prever se um cliente está satisfeito ou insatisfeito com base na sua experiência bancária.

# treinamento dos modelos
def treino_plot(modelos, predictors, target):
    
    # lista para gravar os resultados
    resultados  = []
    nome_modelo = []
    media_ds    = []
    
    # separação do dataset e k-folds (cross-validation)
    kfold = KFold(n_splits = 5, shuffle = True, random_state = 123)

    # treinamento do modelo e gravação dos resultados no dataframe
    for nome, modelo in modelos:
        acc = cross_val_score(modelo, predictors, target, cv = kfold, scoring = 'accuracy')
        resultados.append(acc)
        nome_modelo.append(nome)
        print('{}: {} ({})'.format(nome, round(acc.mean(), 4), round(acc.std(), 6)))
    
    # calcular a média das acurácias dos algortimos treinados
    for i in range(0, len(resultados)):
        media = sum(resultados[i])/len(resultados[i])
        media_ds.append(media)
    
    print('\nMédia das acurácias: ',  round((100 * sum(media_ds)/len(media_ds)), 2))
    print('Desvio padrão das acurácias:', round(np.std(media_ds), 3))
    
    # boxplot para comparar os algoritmos
    fig = plt.figure(figsize = (12,6))
    fig.suptitle('Comparação dos Algoritmos de Classificação')
    ax  = fig.add_subplot(111)
    plt.boxplot(resultados)
    ax.set_xticklabels(nome_modelo)
    plt.show()

# plot roc auc curve
def roc_auc_curve(x, y, otimizador, title = ''):
    # auc score
    fpr, tpr, x = roc_curve(y, otimizador.predict_proba(x)[:, 1])
    roc_auc = round(auc(fpr, tpr), 5)
    
    # plot
    plt.title('AUC {} = {}'.format(title, roc_auc))
    plt.plot(fpr, tpr, 'b')
    plt.plot([0, 1], [0, 1],'r--')
    plt.show()
    
# plot da confusion matrix
def plot_conf_mtx(modelo, x, y, title = ''):
    plot_confusion_matrix(modelo, x, y)
    plt.title(title)
    plt.show()
    
# tuning
def otimizador(x, y):
    # parâmetros
    hyper_xgb = {"learning_rate"    : [0.05, 0.10, 0.15, 0.20, 0.25, 0.30] ,
                 "max_depth"        : [3, 4, 5, 6, 8, 10, 12, 15],
                 "min_child_weight" : [1, 3, 5, 7],
                 "gamma"            : [0.0, 0.1, 0.2 , 0.3, 0.4],
                 "colsample_bytree" : [0.3, 0.4, 0.5 , 0.7]}

    # separação dos dados para testar antes de submeter ao dataset de teste
    x_treino, x_teste, y_treino, y_teste = train_test_split(x, y, test_size = 0.30, random_state = 0)

    # modelo
    modelo_xgb = XGBClassifier()

    # otimizador
    otimizador = RandomizedSearchCV(estimator = modelo_xgb, param_distributions = hyper_xgb,
                                    cv = 10, verbose = True, n_jobs = -1, scoring = 'accuracy',
                                    random_state = 0)

    # treinamento
    otimizador.fit(x_treino, y_treino)

    # melhores parâmetros
    modelo_otimizado = otimizador.best_estimator_
    modelo_otimizado.fit(x_treino, y_treino, eval_set = [(x_teste, y_teste)], eval_metric = 'auc')

    # acurácia
    print('\nAcurácia treino: ', round(100 * modelo_otimizado.score(x_treino, y_treino), 4))
    print('Acurácia teste: ',    round(100 * modelo_otimizado.score(x_teste, y_teste),     4))

    # roc auc curve
    roc_auc_curve(x_treino, y_treino, modelo_otimizado, title = 'treino')
    roc_auc_curve(x_teste,  y_teste,  modelo_otimizado, title = 'teste')
    
    # matriz de confusão
    plot_conf_mtx(modelo_otimizado, x_treino, y_treino, 'Treino')
    plot_conf_mtx(modelo_otimizado, x_teste,  y_teste,  'Teste')
    
    print('Relatório de classificação do teste: \n', classification_report(y_teste, modelo_otimizado.predict(x_teste)))
    
    return modelo_otimizado

4.2 Removendo Variáveis e Registros¶

A variável ID não fornece nenhuma informação para caracterizarmos se o cliente está ou não satisfeito. Por isso, essa variável foi removida. Os IDs do dataset de teste foram gravados em outra variáveis para serem usados posteriormente.

Excluindo o ID, foram encontrado 4807 registros duplicados no dataset de treino. Esses registros duplicados não acrescenta em nada para a construção do modelo preditivo, e por isso todos esses registros foram removidos.

Devido ao alto número de colunas nos datasets, foi investigado a possibilidade de algumas colunas representarem nenhuma informação ou de muitas colunas apresentarem a mesma informação para a caracterização da variável target.

Para a investigação se uma variável não apresenta nenhuma informação, foi observado se para uma mesma coluna não há alteração dos seus valores, ou seja, desvio padrão igual a 0. Essas variáveis foram excluídas do dataset.

Foi possível reduzir 67 colunas.

5.1 Extra Trees Classifier¶

Utilizarei o algoritmo Extra Tree Classifier como indicador das melhores variáveis para o modelo.

O Extra Trees Classifier é um modelo de Machine Learning do conjunto de algoritmos que se baseiam fundamentalmente em árvores de decisão. Esse algortimo, assim como RandomForest, randomiza certas decisões e subconjuntos de dados para minimizar o over-learning dos dados e o overfitting.

Devido a grande diferença entre as duas primeiras posições e as outras variáveis, vou selecionar dois conjuntos de variáveis contendo: duas primeiras posições e as cinco primeiras posições.
